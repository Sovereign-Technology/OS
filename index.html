Repository Structure

Architecture
The Sovereign Stack
Scripts must be loaded in dependency order. Each layer depends on everything below it. The index.js entry point in each app handles service worker registration, stack health checks, and the panic handler.
L0
Cryptographic
sovereign_shamir.js sovereign_security.js
↓ depends on L0
L1
State Machine
sovereign_fsm.js genesis_sw.js (SW)
↓ depends on L0–L1
L2
Transport
transport.js WebRTC · MQTT · CRDT · DHT
↓ depends on L0–L2
L3
OS Kernel
sovereign_kernel.js process · authority · routing
↓ app entry point
L4
App Bootstrap
index.js SW registration · health check · panic handler
↓ application layer
L5
Application
Genesis Node Forge Platform + any sovereign app
Quick Start
Zero install. Open source. Yours.
Sovereign runs entirely as static files. No build step required for local use. Service Workers require HTTP — file:// will not work.
1
Serve over HTTP
Service Workers require a real HTTP origin.
npx serve .  or  python3 -m http.server 8080
2
Open Genesis Node
Navigate to apps/genesis/index.html. The boot screen will verify each layer of the stack as it loads.
3
Create your identity
Click Generate Identity and set a strong passphrase. Your ECDSA P-256 key pair is generated in a Web Worker and encrypted with AES-256-GCM. The private key never exists in the main tab.
4
Back up your recovery shares
Go to Identity Vault → Key Recovery. Export or print your 3-of-5 Shamir shares. Store them in separate physical locations. Without them, a forgotten passphrase means a lost identity.
5
Connect to peers
The transport layer connects to the public relay automatically. Share your DID or QR code to invite peers to your mesh. Use Forge to join the public square.
⚡
Panic shortcut
Press Ctrl+Shift+P at any time to immediately lock the vault and suspend mesh presence. No confirmation required.
